Setup Guide
Get Google Calendar sync working in about 10 minutes.
Part 1 — Host this app online (free)
Google's OAuth requires a live URL. GitHub Pages is free and takes 5 minutes.
1

Create a free GitHub account

Go to github.com and sign up. It's free. You don't need to know how to code.

2

Create a new repository

Click the + icon → New repository. Name it mas-invoices. Set it to Public. Check "Add a README file." Click Create.

3

Upload this file

Click Add file → Upload files. Drag this HTML file in. Name it index.html. Click Commit changes.

4

Enable GitHub Pages

Go to Settings → Pages. Under Source, select main branch. Click Save. Your app will be live at https://[your-username].github.io/mas-invoices in 2 minutes.

Part 2 — Connect Google Calendar API (free)
This lets the app read your calendar events securely.
1

Go to Google Cloud Console

Visit console.cloud.google.com and sign in with any Google account. It's free.

2

Create a new project

Click the project dropdown at the top → New Project. Name it MAS Invoices. Click Create.

3

Enable the Calendar API

Go to APIs & Services → Library. Search for "Google Calendar API." Click it → Enable.

4

Create OAuth credentials

Go to APIs & Services → Credentials. Click Create Credentials → OAuth Client ID. Choose Web application. Under Authorized JavaScript origins, add your GitHub Pages URL. Click Create. Copy your Client ID.

5

Paste your Client ID into this app

Open this HTML file in a text editor. Find the line that says 566546508985-12fom6evof5isvjmea1nqlfm3pj82aun.apps.googleusercontent.com_CLIENT_ID and replace it with the Client ID you just copied. Re-upload to GitHub.

6

Add yourself as a test user

In Google Cloud, go to APIs & Services → OAuth consent screen. Scroll to Test users. Add the Gmail address your calendar is on. Save.
